# 中医聊天机器人系统架构文档

## 1. 系统概述

中医智能助手是一个基于RAG（检索增强生成）+ 关系型数据库的中医知识问答系统，支持针灸、中药、伤寒论、金匮要略等中医知识的智能查询和病情诊断。

### 2.2 模块说明

#### 2.2.1 意图识别模块 (Intent Classifier)

- **功能**: 识别用户查询意图，提取关键实体
- **实现**: 基于规则+关键词匹配
- **输出**: 意图类型、置信度、实体列表

#### 2.2.2 SQL生成模块 (Text2SQL)

- **功能**: 将自然语言转换为SQL查询
- **实现**: 基于规则 + Schema约束
- **支持**: 针灸、中药、伤寒论、金匮要略四张表

#### 2.2.3 RAG检索模块

- **功能**: 基于向量相似度的知识检索
- **实现**: ChromaDB + BGE嵌入模型
- **支持**: 多来源检索、重排序

#### 2.2.4 诊断引擎

- **功能**: 中医辨证分析
- **实现**: 四诊信息提取 + RAG知识检索 + 辨证分析
- **输出**: 证型、病机、治则、推荐方案

## 3. 数据流

### 3.1 查询流程

```
用户输入
    │
    ▼
意图识别 ──> 实体提取
    │
    ├─> 数据库查询意图 ──> Text2SQL ──> 执行SQL ──> 格式化结果
    │
    ├─> 知识查询意图 ──> RAG检索 ──> 结果排序 ──> 生成回复
    │
    └─> 诊断意图 ──> 四诊提取 ──> RAG检索 ──> 辨证分析 ──> 生成诊断
```

### 3.2 数据存储

#### 关系型数据库

| 表名 | 用途 | 主要字段 |
|------|------|----------|
| acupoints | 穴位信息 | name, location, indications, functions |
| herbs | 中药信息 | name, nature, flavor, functions, indications |
| shanghan_formulas | 伤寒论方剂 | name, number, composition, functions |
| jinkui_formulas | 金匮要略方剂 | name, chapter, composition, functions |
| medical_cases | 医案 | symptoms, diagnosis, prescription, result |

#### 向量数据库

| 集合 | 内容 | 用途 |
|------|------|------|
| tcm_knowledge | 所有中医知识 | 统一检索 |
| 按source_type过滤 | 分类检索 | 精确查询 |

## 4. API设计

### 4.1 核心接口

| 接口 | 方法 | 描述 |
|------|------|------|
| /chat | POST | 主聊天接口 |
| /diagnosis | POST | 病情诊断 |
| /query/sql | POST | SQL查询 |
| /knowledge/search | GET | 知识检索 |
| /health | GET | 健康检查 |

## 5. 扩展设计

### 5.1 插件系统

```
plugins/
├── __init__.py
├── pediatrics.py      # 儿科插件
├── gynecology.py      # 妇科插件
└── tuina.py           # 推拿插件
```

### 5.2 新模块添加流程

1. 创建数据表 (db/schema.sql)
2. 添加模型 (db/models.py)
3. 实现查询处理器 (core/或plugins/)
4. 注册意图 (core/intent_classifier.py)
5. 添加API端点 (api/main.py)
6. 更新配置 (config/config.yaml)

## 7. 性能优化

### 7.1 查询优化

- SQL查询添加索引
- 向量检索使用近似最近邻(ANN)
- 结果缓存

### 7.2 缓存策略

- 热门查询结果缓存
- 嵌入向量缓存
- 会话状态缓存
