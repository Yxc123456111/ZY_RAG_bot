# 中药查询功能使用说明

## 功能概述

中药查询功能支持**智能意图识别**，可以自动识别用户输入中的中药名称并查询：

1. **自动意图识别** - 根据语义自动识别中药查询，直接查询 Supabase
2. **草药查询模式** - 专项查询模式，直接查询 Supabase

## 核心特性

### 智能意图识别

在默认聊天窗口下，系统会自动识别用户输入的意图：

- **中药查询** → 直接查询 Supabase 数据库
- **针灸查询** → 调用 API 服务（如果启动）
- **方剂查询** → 调用 API 服务（如果启动）  
- **症状诊断** → 调用 API 服务（如果启动）

**支持的中药查询方式**：
- 直接输入药物名称：`人参`、`黄芪`、`白术`、`苦菜`、`酸枣仁`、`细辛` 等
- 自然语言查询：`人参的功效是什么`、`帮我查一下白术`、`丹砂的药性`

## 快速启动

### 方式一：仅使用智能意图识别（推荐，无需 API）

直接启动桌面客户端：
```bash
python start_desktop.py
```

然后直接输入任何查询，系统会自动识别意图并处理。

### 方式二：使用完整功能（需要 API）

**步骤 1：启动 API 服务**
```bash
python start_api.py
```

**步骤 2：启动桌面客户端**
```bash
python start_desktop.py
```

## 技术实现

### 意图识别流程

```
用户输入
    ↓
意图分类器 (QueryRouter)
    ↓
评分计算（特殊处理：匹配到2字以上关键词时，不归一化分数）
    ↓
├─ 中药查询 (herb_query + is_supabase=True) → 直接查询 Supabase
├─ 针灸查询 (acupuncture_query) → 调用 API
├─ 方剂查询 (shanghan_query/jinkui_query) → 调用 API
└─ 诊断 (diagnosis) → 调用 API
```

### 修复的问题

**问题**：部分药物名称无法识别（如苦菜、酸枣仁、细辛等）

**原因**：
1. 关键词列表不完整
2. 评分算法对长关键词列表归一化过度，导致单个药物名称分数低于阈值

**解决方案**：
1. 扩展关键词列表，添加更多中药名称
2. 修改评分算法：当匹配到2个字符以上的关键词时，不进行归一化处理，避免分数被稀释
